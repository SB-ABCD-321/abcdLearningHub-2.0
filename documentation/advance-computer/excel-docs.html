Cell References

Relative Changes when you copy the formula to another cell
▶
=A1+B1 // If copied down one row, becomes =A2+B2
📋 Real-Life Use
Revenue spreadsheet: =C2*D2 (Price × Qty) — drag down for all rows automatically
Absolute $A$1Locked — never changes when copied
▶
=$B$1*C2 // $B$1 stays fixed; C2 shifts as you copy
📋 Real-Life Use
=C2*$F$1 — Apply a fixed tax rate in F1 to every row's sales value
💡 Tip: Press F4 to cycle between relative, absolute, and mixed references.
Mixed $A1 / A$1Lock just row or just column
▶
=$A2 // Column A locked, row moves =A$2 // Row 2 locked, column moves
📋 Real-Life Use
Multiplication table: =$A3*B$2 — fill right and down for all combinations

From simple sums to conditional aggregations and rounding.

SUMAdd up numbers in a range
▶
SUM(number1, [number2], ...)
ParameterDescription
number1First number or range to add
number2... optionalAdditional numbers or ranges. Up to 255 arguments.
📋 Real-Life: Monthly Sales Total
=SUM(B2:B13) → Adds January through December sales
=SUM(B2:B13, D2:D13) → Adds two product lines together
MonthSales
January45000
February52000
March61000
TOTAL158,000
SUMIFSum values that meet ONE condition
▶
SUMIF(range, criteria, [sum_range])
ParameterDescription
rangeThe cells to evaluate against criteria
criteriaCondition: number, text, expression like ">100", or "*East*"
sum_range optionalCells to sum. If omitted, sums the range itself.
📋 Real-Life: Sales by Region
=SUMIF(C2:C100,"North",D2:D100) → Total sales from North region only
=SUMIF(D2:D100,">10000",D2:D100) → Sum only orders above ₹10,000
=SUMIF(B2:B100,"*Mumbai*",D2:D100) → Sum where city contains "Mumbai"
💡 Wildcards: Use * for any characters, ? for single character, ~* to escape literal asterisk.
SUMIFSSum values that meet MULTIPLE conditions
▶
SUMIFS(sum_range, range1, criteria1, [range2, criteria2], ...)
📋 Real-Life: Region + Product + Quarter Filter
=SUMIFS(E2:E1000, B2:B1000,"North", C2:C1000,"Laptops", D2:D1000,"Q1") → Total laptop sales in Q1 for North region
=SUMIFS(D2:D100, C2:C100,">="&DATE(2024,1,1), C2:C100,"<="&DATE(2024,3,31)) → Sum Jan–Mar 2024
COUNTIF / COUNTIFSCount cells matching conditions
▶
COUNTIF(range, criteria) COUNTIFS(range1, criteria1, [range2, criteria2]...)
📋 Real-Life: HR Attendance / Defect Tracking
=COUNTIF(B2:B200,"Absent") → Count absent employees
=COUNTIFS(C2:C200,"Defect",D2:D200,"High") → Count high-severity defects
=COUNTIF(E2:E100,">="&TODAY()-30) → Orders in last 30 days
AVERAGEIF / AVERAGEIFSConditional average
▶
AVERAGEIF(range, criteria, [average_range])
📋 Real-Life: Average salary by department
=AVERAGEIF(C2:C500,"Engineering",D2:D500) → Avg salary for Engineering dept
=AVERAGEIFS(D2:D500,C2:C500,"Engineering",E2:E500,"Senior") → Avg senior engineer salary
ROUND / ROUNDUP / ROUNDDOWN / MROUNDControl decimal precision
▶
ROUND(number, num_digits) ROUNDUP(number, num_digits) // Always rounds away from zero ROUNDDOWN(number, num_digits) // Always rounds toward zero MROUND(number, multiple) // Round to nearest multiple
📋 Real-Life: Invoice & Pricing
=ROUND(A2*1.18, 2) → Price with 18% GST, rounded to 2 decimals
=ROUNDUP(units/12, 0) → Minimum boxes to order (always round up)
=MROUND(salary, 500) → Round salary to nearest ₹500
SUBTOTALAggregates that ignore filtered rows
▶
SUBTOTAL(function_num, ref1, ...)

Common function_num values: 1=AVERAGE, 2=COUNT, 3=COUNTA, 9=SUM, 11=VAR. Add 100 (e.g., 109) to also ignore manually hidden rows.

📋 Real-Life: Dynamic filtered totals
=SUBTOTAL(9,D2:D1000) → SUM of only visible rows after applying filters — perfect for report footers
=SUBTOTAL(3,A2:A1000) → Count of filtered records

Text Functions

Manipulate, clean, extract, and combine text data.

LEFT / RIGHT / MIDExtract portions of text
▶
LEFT(text, num_chars) RIGHT(text, num_chars) MID(text, start_num, num_chars)
📋 Real-Life: Employee ID / Product Code Parsing
=LEFT(A2,3) → Extract department code from "ENG-00421" → "ENG"
=RIGHT(A2,5) → Extract 5-digit ZIP from address string
=MID(A2,5,4) → Extract year from "INV-2024-0042" → "2024"
LEN / FIND / SEARCHMeasure and locate text
▶
LEN(text) // Count characters FIND(find_text, within) // Case-sensitive position SEARCH(find_text, within) // Case-insensitive, supports wildcards
📋 Real-Life: Split first/last name
=LEFT(A2,FIND(" ",A2)-1) → Extract first name from "Rahul Sharma"
=MID(A2,FIND(" ",A2)+1,LEN(A2)) → Extract last name
SUBSTITUTE / REPLACESwap text within a string
▶
SUBSTITUTE(text, old_text, new_text, [instance_num]) REPLACE(old_text, start_num, num_chars, new_text)
📋 Real-Life: Data Cleaning
=SUBSTITUTE(A2,"-","") → Remove dashes from phone "98-765-43210" → "9876543210"
=SUBSTITUTE(A2," ","_") → Replace spaces with underscores for filenames
=SUBSTITUTE(A2,",","",1) → Remove only the FIRST comma
TRIM / CLEAN / UPPER / LOWER / PROPERClean and format text
▶
TRIM(text) // Remove extra spaces (keeps single spaces between words) CLEAN(text) // Remove non-printable characters UPPER(text) // ALL CAPS LOWER(text) // all lowercase PROPER(text) // First Letter Of Each Word Capitalized
📋 Real-Life: CRM Data Cleanup
=TRIM(CLEAN(A2)) → Fix names imported from another system with weird spacing
=PROPER(TRIM(A2)) → " john DOE " → "John Doe"
CONCATENATE / CONCAT / TEXTJOIN / &Join text strings
▶
& // Simplest join operator CONCAT(text1, text2, ...) // No separator (replaces CONCATENATE) TEXTJOIN(delimiter, ignore_empty, text1, ...) // Join with separator
📋 Real-Life: Build Full Address / Email
=A2&" "&B2 → "Rahul" & "Sharma" → "Rahul Sharma"
=LOWER(A2)&"."&LOWER(B2)&"@company.com" → "rahul.sharma@company.com"
=TEXTJOIN(", ",TRUE,C2:C10) → Join all city names: "Delhi, Mumbai, Chennai"
TEXTFormat numbers as custom text
▶
TEXT(value, format_text)
📋 Real-Life: Report Formatting
=TEXT(A2,"₹#,##0.00") → ₹1,24,500.00
=TEXT(TODAY(),"DD MMM YYYY") → "21 Feb 2026"
="Report for: "&TEXT(A2,"MMMM YYYY") → "Report for: February 2026"
=TEXT(A2,"000000") → Pad 42 to "000042" (for employee IDs)

Date & Time Functions

Calculate deadlines, tenure, working days, and time differences.

TODAY / NOW / DATE / TIMEGet and build dates
▶
TODAY() // Today's date (updates daily) NOW() // Current date AND time DATE(year, month, day) // Build a date from components TIME(hour, minute, second) // Build a time value
📋 Real-Life: Project & Contract Management
=TODAY()+30 → Due date 30 days from today
=DATE(YEAR(A2),MONTH(A2)+3,DAY(A2)) → Add 3 months to any date
DATEDIFCalculate difference between two dates (hidden gem!)
▶
DATEDIF(start_date, end_date, unit) // Units: "Y"=years, "M"=months, "D"=days // "YM"=months ignoring years, "MD"=days ignoring months&years // "YD"=days ignoring years
📋 Real-Life: Employee Tenure & Age
=DATEDIF(B2,TODAY(),"Y") → Years of service (e.g., 7)
=DATEDIF(B2,TODAY(),"Y")&" yrs "&DATEDIF(B2,TODAY(),"YM")&" mos" → "7 yrs 3 mos"
=DATEDIF(DOB,TODAY(),"Y") → Employee age in complete years
⚠️ Note: DATEDIF is undocumented but works in all Excel versions. It does NOT appear in autocomplete — type it manually.
NETWORKDAYS / NETWORKDAYS.INTLCount working days between dates
▶
NETWORKDAYS(start_date, end_date, [holidays]) NETWORKDAYS.INTL(start, end, [weekend], [holidays])
📋 Real-Life: Project Scheduling & SLA Tracking
=NETWORKDAYS(A2,B2,HolidayList) → Working days between start and deadline
=NETWORKDAYS.INTL(A2,B2,1,HolidayList) → Weekend=Sat+Sun; 7=Sunday only
=IF(NETWORKDAYS(A2,TODAY())>5,"⚠️ Overdue","✅ On Track") → SLA status
WORKDAY / WORKDAY.INTL / EDATE / EOMONTHCalculate future/past dates
▶
WORKDAY(start_date, days, [holidays]) // Date N workdays from start EDATE(start_date, months) // Date N months away EOMONTH(start_date, months) // Last day of a month
📋 Real-Life: Contract Expiry & Invoice Due Date
=WORKDAY(TODAY(),5) → Date 5 business days from today (delivery date)
=EDATE(A2,12) → Contract renewal: exactly 12 months from start
=EOMONTH(TODAY(),0) → Last day of current month (invoice cut-off)
=EOMONTH(TODAY(),1)+1 → First day of next month
YEAR / MONTH / DAY / WEEKDAY / WEEKNUMExtract date parts
▶
YEAR(date) MONTH(date) DAY(date) WEEKDAY(date, [return_type]) // 1=Sun-Sat, 2=Mon-Sun WEEKNUM(date, [return_type])
📋 Real-Life: Sales by Month/Quarter/Week
=IF(WEEKDAY(A2,2)>5,"Weekend","Weekday") → Flag weekend transactions
="Q"&INT((MONTH(A2)-1)/3+1) → Convert date to "Q1", "Q2", "Q3", "Q4"
=WEEKNUM(A2,2) → Week number for WoW reporting

Logical Functions

Decision-making, conditions, and error handling in formulas.

IFThe fundamental decision function
▶
IF(logical_test, value_if_true, value_if_false)
📋 Real-Life: Performance Rating
=IF(B2>=90,"Excellent",IF(B2>=75,"Good",IF(B2>=60,"Average","Poor")))
=IF(Stock=0,"🔴 Out of Stock",IF(Stock<10,"🟡 Low Stock","🟢 Available"))
=IF(Sales>Target,"Bonus: "&TEXT(Sales-Target,"₹#,##0"),"No Bonus")
⚠️ Nesting Limit: You can nest IF up to 64 levels deep, but beyond 3–4 it gets hard to read. Use IFS or SWITCH instead.
IFSTest multiple conditions — cleaner than nested IF
▶
IFS(condition1, value1, condition2, value2, ...)
📋 Real-Life: Grade Classification
=IFS(A2>=90,"A",A2>=80,"B",A2>=70,"C",A2>=60,"D",TRUE,"F")
The last TRUE acts as "else" — it always matches if nothing else does.
SWITCHMatch exact values (like a lookup table)
▶
SWITCH(expression, val1, result1, [val2, result2], ..., [default])
📋 Real-Life: Department Code Expansion
=SWITCH(A2,"HR","Human Resources","IT","Information Technology","FIN","Finance","Unknown")
=SWITCH(WEEKDAY(TODAY(),2),6,"Weekend",7,"Weekend","Weekday")
AND / OR / NOT / XORCombine logical conditions
▶
AND(cond1, cond2...) // TRUE if ALL are true OR(cond1, cond2...) // TRUE if ANY is true NOT(condition) // Reverses TRUE/FALSE
📋 Real-Life: Loan Eligibility
=IF(AND(Salary>30000,CreditScore>700,Age>=21),"Eligible","Not Eligible")
=IF(OR(Status="VIP",Purchase>100000),"Priority Queue","Standard")
=IF(AND(Attendance>=75,NOT(OnLeave)),"Eligible for Bonus","Ineligible")
IFERROR / IFNAHandle errors gracefully
▶
IFERROR(value, value_if_error) // Handles ALL errors IFNA(value, value_if_na) // Handles only #N/A
📋 Real-Life: Safe Lookups
=IFERROR(VLOOKUP(A2,Sheet2!A:C,3,0),"Not Found")
=IFERROR(B2/C2,0) → Show 0 instead of #DIV/0! on new rows
=IFNA(XLOOKUP(A2,Table[ID],Table[Name]),"Unknown Employee")
💡 Best Practice: Prefer IFNA over IFERROR for lookups — it only masks #N/A, so other errors (like #REF!) still surface to warn you.
INTERMEDIATE

VLOOKUP & Classic Lookups

The most-used lookup functions — learn these cold.

VLOOKUPLook up a value in the first column, return from another
▶
VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
ParameterDescription
lookup_valueWhat you're looking for (a value, cell, or text)
table_arrayThe table range. Lookup column MUST be first column.
col_index_numColumn number to return (1=first column in range)
range_lookupFALSE = exact match (always use this!). TRUE = approximate (sorted data only)
📋 Real-Life: Employee Directory / Price List
=VLOOKUP(A2,EmployeeTable,4,FALSE) → Get Department for Employee ID
=IFERROR(VLOOKUP(D2,Products!A:E,3,0),"SKU not found") → Safe price lookup
⚠️ Limitations: VLOOKUP cannot look left. It breaks when you insert/delete columns (col_index is a number). For modern use, prefer XLOOKUP or INDEX/MATCH.
A - EMP_IDB - NameC - DeptD - Salary
E001Rahul SharmaEngineering95,000
E002Priya PatelFinance82,000
=VLOOKUP("E002",A:D,3,FALSE)Finance
HLOOKUPLike VLOOKUP but scans across rows
▶
HLOOKUP(lookup_value, table_array, row_index_num, [range_lookup])
📋 Real-Life: Monthly commission rates in a horizontal table
=HLOOKUP("March",CommissionTable,2,FALSE) → Get March's rate from row 2
MATCHFind the position of a value in a range
▶
MATCH(lookup_value, lookup_array, [match_type]) // match_type: 0=exact, 1=less than (sorted asc), -1=greater than (sorted desc)
📋 Real-Life: Find row position for dynamic INDEX
=MATCH("E003",EmployeeIDs,0) → Returns 3 (the row position)
CHOOSEPick one of several values by index number
▶
CHOOSE(index_num, value1, value2, ...)
📋 Real-Life: Quarter from month number
=CHOOSE(MONTH(A2),1,1,1,2,2,2,3,3,3,4,4,4) → Returns quarter number 1–4
=CHOOSE(WEEKDAY(A2),"Sun","Mon","Tue","Wed","Thu","Fri","Sat")
INTERMEDIATE — ADVANCED

XLOOKUP & INDEX/MATCH

Modern, flexible, and more powerful replacements for VLOOKUP.

XLOOKUPThe modern, flexible lookup — replaces VLOOKUP + HLOOKUP + IFERROR
▶
XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])
ParameterDescription
lookup_valueWhat to search for
lookup_arrayColumn or row to search in
return_arrayColumn or row to return — can be to the LEFT!
if_not_found optionalCustom message if not found (replaces IFERROR wrapping)
match_mode optional0=exact, -1=exact or next smaller, 1=exact or next larger, 2=wildcard
search_mode optional1=first to last, -1=last to first, 2=binary ascending, -2=binary descending
📋 Real-Life: Employee Info, Product Pricing
=XLOOKUP(A2,EmpTable[ID],EmpTable[Salary],"Not Found") → Salary by Employee ID
=XLOOKUP(A2,Products[SKU],Products[Price],"SKU Missing",0,-1) → Search from bottom (find latest price)
=XLOOKUP(A2,Table[Name],Table[[Dept]:[Manager]]) → Return multiple columns at once!
💡 XLOOKUP Advantages: Returns value to the left ✓ · Built-in error message ✓ · Search last-to-first ✓ · Exact match by default ✓ · Works with both rows & columns ✓
INDEX / MATCHThe power combo — infinitely flexible lookup
▶
INDEX(array, row_num, [col_num]) // Return value at row/col position MATCH(value, array, 0) // Find position of value // Combined pattern: =INDEX(return_col, MATCH(lookup_val, search_col, 0))
📋 Real-Life: Two-way table lookup
=INDEX(B2:E10,MATCH("E003",A2:A10,0),MATCH("Salary",B1:E1,0)) → Find value at intersection of Employee and column header
=INDEX(A2:A100,MATCH(MAX(D2:D100),D2:D100,0)) → Find name of top-earning employee
=INDEX(A:A,MATCH(1,(B:B="Delhi")*(C:C="Q1"),0)) → Match on TWO criteria (Ctrl+Shift+Enter)
XMATCHModern MATCH with extra options
▶
XMATCH(lookup_value, lookup_array, [match_mode], [search_mode])
📋 Real-Life: Find rank or position
=XMATCH(A2,SortedScores,-1) → Find position with approximate match for grade bands
=XMATCH("*Mumbai*",CityList,2) → Wildcard search for "Mumbai" anywhere in cell
ADVANCED

Array Formulas & SUMPRODUCT

Do calculations across entire ranges without helper columns.

SUMPRODUCTMultiply arrays and sum — the most versatile advanced function
▶
SUMPRODUCT(array1, [array2], ...) // Multiplies corresponding elements, then sums all results // Use -- to convert TRUE/FALSE to 1/0 for conditions
📋 Real-Life: Weighted Averages, Multi-Condition Sums
=SUMPRODUCT(B2:B100,C2:C100) → Total revenue: qty × price for each row, summed
=SUMPRODUCT((A2:A100="North")*(C2:C100="Q1"),D2:D100) → Conditional sum (no CSE needed!)
=SUMPRODUCT(B2:B10*C2:C10)/SUM(C2:C10) → Weighted average (grade × weight / total weight)
=SUMPRODUCT((LEN(A2:A100)-LEN(SUBSTITUTE(A2:A100,",","")))+1) → Count comma-separated items across all cells
💡 Why SUMPRODUCT? It works without Ctrl+Shift+Enter, handles text conditions natively, and can replace COUNTIFS/SUMIFS when you need extra flexibility.
CSE Array Formulas { }Legacy arrays entered with Ctrl+Shift+Enter
▶
// Enter with Ctrl+Shift+Enter — Excel adds { } automatically // In Excel 365, most CSE formulas can just be entered normally {=SUM(IF(A2:A100="North", B2:B100*C2:C100))} // Sum of (price × qty) for North region only
📋 Real-Life: Count unique values
{=SUM(1/COUNTIF(A2:A100,A2:A100))} → Count distinct values in a range
{=MAX(IF(B2:B100="Sales",C2:C100))} → Max salary in Sales department
AGGREGATESUBTOTAL's powerful sibling — ignores errors AND hidden rows
▶
AGGREGATE(function_num, options, array, [k]) // Options: 5=ignore hidden, 6=ignore errors, 7=ignore both // Functions include: 14=LARGE, 15=SMALL, 17=MODE, 19=PERCENTILE
📋 Real-Life: Top-N sales, robust stats
=AGGREGATE(14,6,D2:D100,1) → Largest value ignoring errors (#N/A in some cells)
=AGGREGATE(15,7,D2:D100,3) → 3rd smallest value, ignore hidden rows and errors
ADVANCED — Excel 365 / 2021+

Dynamic Array Functions

Revolutionary functions that spill results automatically into multiple cells.

FILTERExtract rows matching a condition — dynamic and live
▶
FILTER(array, include, [if_empty])
📋 Real-Life: Dynamic department list, filtered orders
=FILTER(A2:D100,C2:C100="North","No data") → Show only North region rows — updates live!
=FILTER(A2:D100,(C2:C100="North")*(D2:D100>50000)) → Multiple conditions: North AND Sales > 50K
=FILTER(A2:D100,(C2:C100="North")+(C2:C100="South")) → OR: North or South region
SORT / SORTBYReturn a sorted array dynamically
▶
SORT(array, [sort_index], [sort_order], [by_col]) SORTBY(array, by_array1, [order1], [by_array2], ...)
📋 Real-Life: Live leaderboard, sorted report
=SORT(A2:C100,3,-1) → Sort by column 3, descending (top sales first)
=SORTBY(A2:C100,C2:C100,-1,B2:B100,1) → Sort by sales desc, then by name asc
=SORT(FILTER(A2:C100,C2:C100>10000),3,-1) → Filter then sort in one formula!
UNIQUEExtract a distinct list from a range
▶
UNIQUE(array, [by_col], [exactly_once])
📋 Real-Life: Build dynamic dropdown source
=UNIQUE(B2:B500) → Unique list of cities from 500-row dataset
=SORT(UNIQUE(C2:C200)) → Sorted unique department names for a dropdown
=COUNTA(UNIQUE(A2:A100)) → Count distinct customers
SEQUENCEGenerate number series automatically
▶
SEQUENCE(rows, [cols], [start], [step])
📋 Real-Life: Auto-number, date series
=SEQUENCE(12,1,DATE(2024,1,1),30) → 12 monthly dates starting Jan 2024
=SEQUENCE(10) → Numbers 1 to 10
=TEXT(SEQUENCE(12,1,DATE(2024,1,1),28),"MMM YY") → Month labels: Jan 24, Feb 24...
TOCOL / TOROW / VSTACK / HSTACK / TEXTSPLIT / TEXTBEFORE / TEXTAFTERExcel 2023+ advanced manipulation
▶
TOCOL(array) // Flatten 2D range into one column VSTACK(array1, array2) // Stack ranges vertically HSTACK(array1, array2) // Stack ranges horizontally TEXTSPLIT(text, col_delim, [row_delim]) // Split text into array TEXTBEFORE(text, delimiter) // Text before first delimiter TEXTAFTER(text, delimiter) // Text after first delimiter
📋 Real-Life: Combine multiple data tables
=VSTACK(Jan_Data,Feb_Data,Mar_Data) → Combine 3 month sheets into one table
=TEXTBEFORE(A2,"@") → Extract username from email "rahul@company.com" → "rahul"
=TEXTSPLIT(A2,",") → Split "Delhi, Mumbai, Chennai" into 3 cells
ADVANCED

Financial Functions

Loan calculations, investment analysis, depreciation, and cash flow.

PMTCalculate periodic loan payment
▶
PMT(rate, nper, pv, [fv], [type]) // rate: interest rate per period (annual rate ÷ 12 for monthly) // nper: total number of payments // pv: present value (loan amount) — enter as negative // fv: future value after last payment (default 0) // type: 0=end of period, 1=beginning
📋 Real-Life: Home Loan / Car EMI Calculator
=PMT(8.5%/12, 240, -5000000) → Monthly EMI for ₹50L home loan at 8.5% for 20 years
=PMT(B1/12, B2*12, -B3) → Dynamic EMI calculator (rate in B1, years in B2, amount in B3)
⚠️ Sign Convention: PMT returns a negative number (cash outflow). Wrap in ABS() or enter PV as negative to get a positive result.
NPV / XNPVNet Present Value of cash flows
▶
NPV(rate, value1, [value2], ...) // Assumes equal time periods XNPV(rate, values, dates) // Unequal time periods (more accurate)
📋 Real-Life: Project investment decision
=NPV(10%,B2:B7)-B1 → NPV of 6-year project (subtract initial investment in B1)
=IF(NPV(10%,B2:B7)-B1>0,"✅ Accept Project","❌ Reject Project")
=XNPV(10%,C2:C10,D2:D10) → NPV with actual dates (handles irregular cash flows)
IRR / XIRRInternal Rate of Return
▶
IRR(values, [guess]) XIRR(values, dates, [guess])
📋 Real-Life: Compare multiple investments
