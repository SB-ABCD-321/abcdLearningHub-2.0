🌐 HTML → What is HTML?
🌐 What is HTML?
HTML stands for HyperText Markup Language. It's the language every single website in the world is built with. Think of it as LEGO bricks — you snap pieces together to build pages! 🧱
1

HTML is Like a Sandwich 🥪

Easy

Most HTML tags come in pairs — an opening tag and a closing tag. The closing tag has a / in it. The content goes in the middle, like the filling!

HTML Example
<!-- Opening tag → Content → Closing tag -->
<p>I am a paragraph! 🍔</p>

<h1>I am a big heading! 🎉</h1>

<strong>I am bold text!</strong>

<em>I am italic text!</em>
🌟 Fun Fact: HTML was invented in 1991 by Tim Berners-Lee, a scientist in Switzerland. He wanted a way to share documents with other scientists. It turned into the World Wide Web!
2

The Basic HTML Template 📄

Easy

Every HTML file starts with the same basic structure. Think of it as the rules of the game — you need them before you play!

index.html
<!DOCTYPE html>     <!-- Tells browser: "I'm an HTML file!" -->
<html lang="en">       <!-- Root element + language -->

  <head>             <!-- Hidden info (not shown on page) -->
    <meta charset="UTF-8">        <!-- Support all letters -->
    <meta name="viewport" content="width=device-width">
    <title>My Awesome Page 🌟</title>
  </head>

  <body>             <!-- Everything VISIBLE goes here! -->
    <h1>Hello World! 🌍</h1>
    <p>This is my very first webpage!</p>
  </body>

</html>
💡 VS Code Shortcut: In VS Code, type ! and press Tab — it automatically creates the full template for you! This is called an Emmet shortcut! 🚀
3

HTML Attributes 🔧

Easy

Attributes give extra information to a tag. They live inside the opening tag and have a name="value" format.

HTML Attributes
<!-- The tag has an attribute -->
<a href="https://google.com">Go to Google</a>
<!--   ↑ attribute name  ↑ attribute value -->

<img src="cat.jpg" alt="A cute cat" width="200">

<p class="intro" id="first-para">Hello!</p>
🎯 Quick Check!
What does the DOCTYPE html line tell the browser?

1

Heading Tags h1–h6 🎯

Easy

Headings organize your content like chapters in a book. h1 is the biggest and most important — only use ONE per page! h2–h6 get progressively smaller.

Headings
<h1>The Main Title of the Page 📖</h1>

<h3>A Sub-section</h3>
<h4>A smaller heading</h4>
<h5>Even smaller</h5>
<h6>The smallest heading</h6>
⚠️ Important Rule: Only ever use <h1> ONCE on a page — for the main title. This helps search engines (like Google) understand your page! 🔍

🌐 HTML → Page Structure
🏗️ Page Structure
Well-organized HTML is easier to read and maintain. Let's learn how to structure pages like real professionals! 🏠
1

Block vs Inline Elements 📐

Medium

Block elements take up the full width of the page — they start on a new line. Inline elements only take up as much space as they need and sit next to other content.

Block vs Inline
<!-- BLOCK elements (each on its own line) -->
<div>I'm a block element</div>
<p>I'm also a block element</p>
<h1>Me too!</h1>

<!-- INLINE elements (sit inside text) -->
<p>My name is <strong>Alex</strong> and I like
<span style="color:red">coding</span>!</p>
🌐 HTML → Links & Images
🔗 Links & Images
Links connect pages together — that's literally how the "web" works! Images make pages come alive! 🖼️
1

Creating Links 🔗

Easy

The <a> tag (anchor) makes clickable links. The href attribute says where to go. You can link to websites, other pages, or even different sections of the same page!

HTML Links
<!-- External link -->
<a href="https://www.google.com">Go to Google 🔍</a>

<!-- Open in a new tab (don't close my page!) -->
<a href="https://youtube.com" target="_blank">YouTube 🎥</a>

<!-- Link to another page in your project -->
<a href="about.html">About Me Page</a>

<!-- Jump to a section on the same page -->
<a href="#contact">Jump to Contact 📍</a>

<!-- Link an image (click image to go somewhere) -->
<a href="https://nasa.gov">
  <img src="rocket.png" alt="Rocket 🚀">
</a>
2

Adding Images 🖼️

Easy

Images use the <img> tag. It's a self-closing tag — no closing tag needed! Always add alt text: it helps people with screen readers and when images fail to load.

HTML Images
<!-- Image from internet -->
<img
  src="https://placekitten.com/300/200"
  alt="A cute kitten 🐱"
  width="300"
  height="200"
>

<!-- Image from your project folder -->
<img src="images/my-photo.jpg" alt="My photo">
💡 Pro Tip: Create an images/ folder inside your project folder and keep all images there. Then reference them as src="images/filename.jpg" — clean and organized! 📁

🌐 HTML → Semantic HTML
🧩 Semantic HTML
Semantic tags describe what they contain — not just how they look. They make your code more readable, help search engines, and help people with disabilities. Use them always! ♿
1

Semantic Tags Explained 🏠

Medium
Semantic HTML Layout
<header>   <!-- Logo, site name, top navigation -->
  <h1>🌟 My Awesome Site</h1>
  <nav>
    <a href="index.html">Home</a>
    <a href="about.html">About</a>
    <a href="contact.html">Contact</a>
  </nav>
</header>

<main>       <!-- The main content of the page -->

  <section>  <!-- A group of related content -->
    <h2>Latest News 📰</h2>
    <article> <!-- A standalone piece of content -->
      <h3>Coding is Fun!</h3>
      <p>Today I learned HTML...</p>
    </article>
  </section>

  <aside>    <!-- Sidebar info (tips, ads, links) -->
    <h4>💡 Coding Tips</h4>
    <p>Practice every day!</p>
  </aside>

</main>

<footer>   <!-- Bottom of page: credits, links -->
  <p>Made with ❤️ by a Kid Coder! 2025</p>
</footer>
🌟 Why Semantic HTML Matters: Google reads your semantic tags to understand what your page is about. If you use <article> and <header>, Google ranks your page higher in search results! 📈

🎨 CSS → What is CSS?
🎨 What is CSS?
CSS — Cascading Style Sheets — controls how your HTML looks. If HTML is the skeleton, CSS is the clothes, hair, and makeup! Without CSS, websites would all be boring black text on white backgrounds. 😴
1

CSS Syntax Rules 🔤

Easy

CSS is made of rules. Each rule has a selector (who to style) and a declaration block with properties and values.

CSS Rule Anatomy
/* selector { property: value; } */

h1 {                    /* selector: target h1 tags */
  color: hotpink;        /* property: value */
  font-size: 48px;
  text-align: center;
}

p {
  color: #333;
  line-height: 1.6;    /* Space between lines */
  font-size: 16px;
}
🌟 "Cascading" Means: Styles flow downward like a waterfall. If you style <body>, all children inherit those styles unless you override them. The more specific rule always wins! 🌊
2

3 Ways to Add CSS 🔗

Easy
Ways to add CSS
<!-- 1. EXTERNAL file (BEST practice!) -->
<link rel="stylesheet" href="style.css">

<!-- 2. Internal <style> tag in <head> -->
<style>
  body { background: lightblue; }
</style>

<!-- 3. Inline (avoid this! Hard to manage) -->
<p style="color: red; font-size: 20px;">Red text</p>
🚀 Pro Tip: Always use Method 1 — a separate style.css file. It keeps your HTML and CSS separate, making your code much cleaner and easier to update! 🧹

🎨 CSS → Selectors
🎯 CSS Selectors
Selectors tell CSS which elements to style. Master these and you can style any element exactly how you want! 🫵
1

All Selector Types 🎯

Medium
CSS Selectors
/* Tag: targets ALL elements of this type */
h1 { color: red; }
p  { color: gray; }

/* Class: targets elements with class="name" */
.highlight { background: yellow; }
.big-text   { font-size: 28px; }

/* ID: targets ONE specific element */
#hero-title { font-size: 64px; }

/* Multiple selectors at once */
h1, h2, h3 { font-family: cursive; }

/* Descendant: p inside a div */
div p { color: blue; }

/* Hover state */
button:hover {
  background: hotpink;
  cursor: pointer;
}

/* Focus state (when clicked / keyboard) */
input:focus {
  border-color: blue;
  outline: 3px solid lightblue;
}
💡 Class vs ID: Use class when many elements share the same style. Use ID for one-of-a-kind elements. A good rule: 90% of the time, use class! 😊
🎨 CSS → Colors & Fonts
🌈 Colors & Fonts
Colors and fonts are the most fun part of CSS! Make your pages look truly unique and beautiful! 💅
1

Color Methods 🌈

Easy
CSS Colors
div {
  /* 1. Named colors (easy to read!) */
  color: tomato;
  color: dodgerblue;
  color: hotpink;

  /* 2. HEX codes (6 digits: R R G G B B) */
  color: #FF6B35;      /* Orange */
  color: #4ECDC4;      /* Teal */
  color: #fff;         /* White (short) */

  /* 3. RGB (Red Green Blue, 0-255) */
  color: rgb(255, 0, 128);

  /* 4. RGBA (with transparency 0-1) */
  color: rgba(255, 0, 128, 0.5);  /* 50% transparent */

  /* 5. Gradient backgrounds! */
  background: linear-gradient(to right, #FF6B35, #FFD93D);
  background: radial-gradient(circle, purple, blue);
}
🌈 Fun Fact: There are 16.7 million different colors you can make with HEX codes! That's more than the number of colors humans can actually see! 🤯
2

Fonts & Typography 🔤

Easy
CSS Typography
body {
  font-family: 'Nunito', Arial, sans-serif;  /* Fallbacks! */
  font-size: 16px;
  font-weight: 400;    /* 100 thin → 900 black */
  line-height: 1.6;    /* Space between lines */
  letter-spacing: 0.5px;
}

h1 {
  font-weight: 900;    /* Extra bold! */
  text-transform: uppercase;
  text-shadow: 2px 2px 10px rgba(0,0,0,0.3);
}

/* Google Fonts — add this in HTML head first:
<link href="https://fonts.googleapis.com/css2?
family=Fredoka+One&family=Nunito:wght@400;700"
rel="stylesheet"> */

.fancy { font-family: 'Fredoka One', cursive; }
3

CSS Variables 🔧

Medium

CSS Variables (custom properties) let you store values and reuse them. Change the variable once, and it updates everywhere! This is like a magic paintbrush! 🪄

CSS Variables
/* Define variables in :root (global) */
:root {
  --primary: #FF6B35;
  --secondary: #4ECDC4;
  --text: #333;
  --radius: 12px;
}

/* Use them with var() */
button {
  background: var(--primary);
  border-radius: var(--radius);
  color: white;
}

h1 { color: var(--primary); }
p  { color: var(--text); }
🎨 CSS → Box Model
📦 The Box Model
Every HTML element is a rectangular box. Once you understand the box model, spacing in CSS makes total sense! 📦
1

Box Model Layers 📦

Medium

Imagine a gift box: the present is content, the bubble wrap is padding, the box itself is border, and the space between boxes on a shelf is margin!

CSS Box Model
.card {
  width: 300px;            /* Content width */
  height: 200px;           /* Content height */

  padding: 20px;           /* Space INSIDE (all sides) */
  padding-top: 10px;       /* Just top padding */
  padding: 10px 20px;     /* top/bottom  left/right */

  border: 3px solid hotpink; /* The outline */
  border-radius: 16px;    /* Rounded corners! */

  margin: 20px;            /* Space OUTSIDE */
  margin: 0 auto;          /* Center horizontally! */

  /* box-sizing: border-box means padding is
     INCLUDED in the width — use this always! */
  box-sizing: border-box;
}

/* Apply box-sizing to EVERYTHING (best practice) */
* { box-sizing: border-box; }
💡 Always add this! Put * { box-sizing: border-box; } at the TOP of every CSS file. It makes sizing elements much more predictable! This is what every professional uses! 💼
🎨 CSS → Flexbox
📐 Flexbox Layout
Flexbox makes arranging elements in a row or column super easy! Before flexbox, centering things was a nightmare. Now it's one line of code! 🎉
1

Flexbox Fundamentals 📐

Medium
Flexbox
/* Parent becomes a flex container */
.container {
  display: flex;

  /* Direction */
  flex-direction: row;         /* default (horizontal) */
  flex-direction: column;      /* vertical */

  /* Horizontal alignment */
  justify-content: center;     /* center items */
  justify-content: space-between; /* spread out */
  justify-content: flex-end;   /* push to right */

  /* Vertical alignment */
  align-items: center;        /* center vertically */
  align-items: flex-start;    /* top aligned */

  gap: 16px;                   /* space between items */
  flex-wrap: wrap;             /* wrap to next line */
}

/* Children can grow/shrink */

    options: ["Happy Text Making Language", "HyperText Markup Language", "Hyper Tool Making Language"],
    correct: 1   // index of correct answer
  },
  {

let score = 0;

function showQuestion() {
  const q = questions[currentQuestion];
  document.querySelector("#question").textContent = q.question;

  const optionsDiv = document.querySelector("#options");
  optionsDiv.innerHTML = "";

  q.options.forEach((option, index) => {
    const btn = document.createElement("button");
    btn.textContent = option;
    btn.addEventListener("click", () => checkAnswer(index));
    optionsDiv.appendChild(btn);
  });
}

function checkAnswer(selected) {
  const q = questions[currentQuestion];
  if (selected === q.correct) {
    score++;
    alert("✅ Correct!");
  } else {
    alert("❌ Wrong! The answer was: " + q.options[q.correct]);
  }
  currentQuestion++;
  if (currentQuestion < questions.length) {
    showQuestion();
  } else {
    document.querySelector("#question").textContent =
      `Game Over! Score: ${score}/${questions.length} 🏆`;
  }
}

let todos = JSON.parse(localStorage.getItem("todos")) || [];

function addTodo() {
  const input = document.querySelector("#todo-input");
  const text = input.value.trim();
  if (!text) return;  // Don't add empty todos

  todos.push({ text: text, done: false, id: Date.now() });
  input.value = "";
  saveTodos();
  renderTodos();
}

function toggleDone(id) {
  todos = todos.map(t => t.id === id ? {...t, done: !t.done} : t);
  saveTodos();
  renderTodos();
}

function deleteTodo(id) {
  todos = todos.filter(t => t.id !== id);
  saveTodos();
  renderTodos();
}

function saveTodos() {
  localStorage.setItem("todos", JSON.stringify(todos));
}

function renderTodos() {
  const list = document.querySelector("#todo-list");
  list.innerHTML = todos.map(todo => `
    <li class="${todo.done ? 'done' : ''}">
      <span onclick="toggleDone(${todo.id})">${todo.text}</span>
      <button onclick="deleteTodo(${todo.id})">🗑️</button>
    </li>
  `).join("");
}

renderTodos();  // Load saved todos on page open

    display = display.slice(0, -1);
  } else {
    display += value;
  }
  document.querySelector("#display").textContent = display || "0";
}

const API_KEY = "YOUR_FREE_API_KEY"; // Get from openweathermap.org

async function getWeather(city) {
  const url = `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${API_KEY}&units=metric`;

  try {
    const response = await fetch(url);
    const data = await response.json();

    console.log(data);

    document.querySelector("#city").textContent = data.name;
    document.querySelector("#temp").textContent = `${Math.round(data.main.temp)}°C`;
    document.querySelector("#desc").textContent = data.weather[0].description;

    // Change background based on weather
    const weather = data.weather[0].main;
    document.body.style.background = weather === "Clear" ?
      "linear-gradient(135deg, #87CEEB, #FFD700)" :
      "linear-gradient(135deg, #4a4a6a, #6a8fa0)";

  } catch (error) {
    alert("City not found! Try again 🌍");
  }
}
